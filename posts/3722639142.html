3.1 安装 webpack 工具

  1. 安装三个包： webpack webpack-cli webpack-dev-server

命令： npm i webpack webpack-cli webpack-dev-server -D

  1. 根据 开发 / 生产 两个环境，配置不同的 webpack 配置文件。配置文件放置在 ./build 文件夹下。
  2. webpack.config.js：是所有文件的入口。
  3. webpack.base.js：是 开发 / 生产 环境的公共配置文件。
  4. webpack.dev.js：开发环境配置文件。
  5. webpack.pro.js：生产环境配置文件。

3.2 对各个 webpack 配置文件的简单配置与解释

  1. webpack.base.js：公共环境配置文件
  2. 利用 module.rules ,这些规则能够修改模块的创建方式。 这些规则能够对模块(module)应用 loader，或者修改解析器(parser)。*外链
  3. 在 plugins 中添加 html-webpack-plugin 插件，HtmlWebpackPlugin 简化了 HTML 文件的创建，以便为你的 webpack 包提供服务。*外链
  1. webpack.dev.js：开发环境的配置文件

拆解 devtool 中的字符含义：

eval 打包后的模块都使用 eval() 执行，行映射可能不准；不产生独立的 map 文件
cheap map 映射只显示行不显示列，忽略源自 loader 的 source map
inline 映射文件以 base64 格式编码，加在 bundle 文件最后，不产生独立的 map 文件
module 增加对 loader source map 和第三方模块的映射

  1. webpack.pro.js：生产环境的配置文件

CleanWebpackPlugin 的作用是，在成功构建以后，帮助我们清空构建目录，因为有的时候为了避免缓存，我们需要在文件后加入 hash 。这样再多次构建后就会产生很多无用的旧构建文件，通过这个插件就能自动的帮我们清空这些目录。

  1. webapck.config.js：配置文件的入口文件

WebpackMerge 的作用就是将多个配置文件合并为一个配置文件。
